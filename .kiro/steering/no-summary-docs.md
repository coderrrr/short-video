# 避免创建总结文档

## 规则

**不要在每次操作后创建总结性的Markdown文档。**

## 说明

- ❌ 不要创建类似 `XXX_SUMMARY.md`、`XXX_REPORT.md`、`XXX_GUIDE.md` 等总结文档
- ❌ 不要创建重复说明已完成工作的文档
- ✅ 只在用户明确要求时创建文档
- ✅ 直接在回复中简洁说明完成的工作即可

## 例外情况

以下情况可以创建文档：
1. 用户明确要求创建文档
2. 项目必需的配置文件（如 README.md、SETUP.md）
3. 重要的技术文档（如 API文档、架构设计）
4. 测试脚本或工具脚本（.sh 文件）

## 原则

保持项目目录整洁，避免创建不必要的文档文件。
