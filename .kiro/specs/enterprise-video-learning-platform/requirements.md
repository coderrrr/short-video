# 需求文档

## 简介

本文档规定了企业内部短视频平台的需求 - 这是一个自建的移动优先内容共享系统，旨在鼓励员工在企业内部进行视频创作，既可以分享工作知识和经验，也可以分享美好生活，从而丰富企业文化。该平台通过员工生成内容、智能推荐和与岗位角色及技能相匹配的结构化学习路径，促进组织内部知识传递、生活分享和文化建设。

### 技术架构约束

- **C端（用户端）**：以H5方式集成在企业现有App中，不提供独立的PC Web界面
- **管理端**：提供PC Web管理后台界面

## 术语表

- **平台**：企业内部短视频平台系统
- **员工用户**：企业内部观看和消费视频内容的员工
- **员工创作者**：企业内部制作和上传视频内容的员工
- **KOL（关键意见领袖）**：企业内部具有特殊内容创作权限的指定专家员工
- **管理员**：企业内部具有内容审核和系统配置权限的平台管理者
- **内容**：员工创作的带有相关元数据（标题、描述、标签等）的视频项目，包括工作知识分享和生活美好瞬间
- **草稿**：已保存但尚未提交审核的内容
- **审核状态**：审批工作流中的内容状态（草稿、审核中、已通过、已驳回、已发布、已下架）
- **标签体系**：多维分类框架，包括角色标签、主题标签、形式标签、质量标签和推荐标签
- **合集**：精选的相关内容项目组
- **专题**：内容项目的主题分组
- **学习计划**：与岗位角色和技能要求相匹配的结构化内容序列
- **移动端（C端）**：以H5方式集成在企业现有App中的面向最终用户的界面，不包含独立的PC Web界面
- **后台系统（管理端）**：用于平台管理的PC Web管理界面

## 需求

### 需求 1：视频内容上传

**用户故事：** 作为员工创作者，我想从移动设备捕获和上传视频内容，以便在企业内部与同事分享知识和经验。

#### 验收标准

1. 当创作者选择相机拍摄选项时，平台应激活设备相机进行直接视频录制
2. 当创作者选择本地导入选项时，平台应显示设备相册以供视频选择
3. 当创作者完成视频选择或录制时，平台应接受常见格式的视频文件（MP4、MOV、AVI）

4. 当视频文件超过平台大小限制时，平台应拒绝上传并显示包含最大允许大小的错误消息
5. 当在Native App环境下且视频文件大小超过压缩阈值（如50MB）时，平台应自动压缩视频以减小文件大小并保持可接受的质量（Web环境下不支持压缩）
6. 当视频上传开始时，平台应向创作者显示上传进度

### 需求 2：视频元数据管理

**用户故事：** 作为员工创作者，我想为视频添加描述性信息，以便企业内部其他员工能够发现和理解我分享的内容。

#### 验收标准

1. 当创作者上传视频时，平台应要求创作者提供标题
2. 当创作者提供元数据时，平台应接受最多500个字符的描述字段
3. 当创作者选择内容类型时，平台应显示预定义的内容类型选项（包括工作知识、生活分享、企业文化等类别）
4. 当创作者添加自定义标签时，平台应接受由分隔符分隔的字母数字标签值
5. 当创作者保存元数据时，平台应验证标题不为空

### 需求 3：视频封面图片选择

**用户故事：** 作为员工创作者，我想为视频设置吸引人的封面图片，以便吸引企业内部同事观看。

#### 验收标准

1. 当创作者访问封面选择时，平台应以固定间隔提取并显示视频帧
2. 当创作者选择视频帧时，平台应将该帧设置为视频封面图片
3. 当创作者选择本地相册选项时，平台应显示设备相册以供图片选择
4. 当创作者选择本地图片时，平台应验证图片格式（JPEG、PNG）和尺寸


### 需求 4：视频编辑功能

**用户故事：** 作为员工创作者，我想在发布前编辑视频，以便提高分享内容的质量和相关性。

#### 验收标准

1. 当创作者选择视频裁剪时，平台应显示带有开始和结束标记的时间轴界面
2. 当创作者调整裁剪标记时，平台应实时更新视频预览
3. 当创作者确认裁剪时，平台应将视频处理到指定时长
4. 当创作者访问音量控制时，平台应显示带有百分比指示器的音量滑块
5. 当创作者调整音量滑块时，平台应相应修改视频音频级别

### 需求 5：内容草稿管理

**用户故事：** 作为员工创作者，我想保存进行中的创作工作，以便稍后完成和发布到企业内部平台。

#### 验收标准

1. 当创作者选择保存为草稿时，平台应以"草稿"状态存储视频和元数据
2. 当创作者访问草稿时，平台应显示所有状态为"草稿"的内容项目
3. 当创作者编辑草稿时，平台应加载先前保存的视频和元数据
4. 当创作者删除草稿时，平台应从存储中删除内容和相关文件


### 需求 6：内容提交和审核

**用户故事：** 作为员工创作者，我想提交内容进行企业内部审核，以便在批准后发布到企业平台供同事观看。

#### 验收标准

1. 当创作者提交内容时，平台应将内容状态更改为"审核中"
2. 当内容状态更改为"审核中"时，平台应通知指定的审核员
3. 当内容处于审核中时，平台应阻止创作者编辑内容
4. 当管理员批准内容时，平台应将状态更改为"已发布"并使其对用户可见
5. 当管理员拒绝内容时，平台应将状态更改为"已驳回"并向创作者通知拒绝原因



### 需求 9：按分类发现内容

**用户故事：** 作为员工用户，我想按分类浏览企业内部视频内容，以便找到与特定主题相关的知识分享。

#### 验收标准

1. 当用户访问分类导航时，平台应显示所有活动的内容分类
2. 当用户选择分类时，平台应显示所有标记有该分类的已发布内容
3. 当分类包含子分类时，平台应显示层次结构
4. 当用户导航到子分类时，平台应过滤内容以仅显示该子分类中的项目

### 需求 10：个性化内容推荐

**用户故事：** 作为员工用户，我想看到根据我的岗位和兴趣定制的推荐内容，以便高效地发现企业内部相关知识和文化内容。

#### 验收标准

1. 当用户访问推荐信息流时，平台应显示由推荐算法排名的内容
2. 当计算推荐时，平台应考虑用户的角色标签、观看历史和互动模式
3. 当用户与内容互动时，平台应更新用户的偏好配置文件
4. 当新内容发布时，平台应评估其是否包含在相关用户的推荐信息流中
5. 当用户刷新推荐信息流时，平台应提供更新的内容建议


### 需求 11：关注系统

**用户故事：** 作为员工用户，我想关注企业内部我认为有价值的创作者同事，以便轻松访问他们分享的新内容。

#### 验收标准

1. 当用户在创作者个人资料上选择关注时，平台应将该创作者添加到用户的关注列表
2. 当用户访问关注信息流时，平台应按时间倒序显示关注创作者的内容
3. 当关注的创作者发布新内容时，平台应将其添加到关注者的信息流
4. 当用户取消关注创作者时，平台应从关注列表中删除该创作者并停止在关注信息流中显示其内容
5. 当用户查看创作者个人资料时，平台应指示用户当前是否正在关注该创作者

### 需求 12：内容搜索

**用户故事：** 作为员工用户，我想使用关键词搜索企业内部视频内容，以便快速找到特定的知识和信息。

#### 验收标准

1. 当用户输入搜索查询时，平台应在内容标题、描述和标签中搜索匹配的术语
2. 当返回搜索结果时，平台应按相关性排名显示它们
3. 当搜索查询包含多个词时，平台应将它们视为具有OR逻辑的单独关键词
4. 当没有结果匹配查询时，平台应显示指示未找到内容的消息
5. 当用户清除搜索查询时，平台应返回到先前的视图


### 需求 13：多维内容筛选

**用户故事：** 作为用户，我想通过多个标准筛选内容，以便将结果缩小到我的特定需求。

#### 验收标准

1. 当用户访问筛选选项时，平台应显示包括岗位、技能和内容类型的筛选类别
2. 当用户选择多个筛选器时，平台应使用AND逻辑应用它们
3. 当应用筛选器时，平台应仅显示匹配所有选定标准的内容
4. 当用户清除筛选器时，平台应恢复未筛选的内容视图
5. 当没有内容匹配筛选器组合时，平台应显示指示无结果的消息

### 需求 14：视频播放控制

**用户故事：** 作为用户，我想要全面的播放控制，以便以我喜欢的节奏和风格消费内容。

#### 验收标准

1. 当用户点击视频时，平台应在播放和暂停状态之间切换
2. 当用户拖动进度条时，平台应跳转到相应的视频位置
3. 当用户垂直滑动时，平台应加载并播放信息流中的下一个视频
4. 当用户选择播放速度时，平台应提供速度选项（0.5x、0.75x、1x、1.25x、1.5x、2x）
5. 当播放速度改变时，平台应调整视频播放速率同时保持音频音调


### 需求 15：内容互动 - 收藏

**用户故事：** 作为用户，我想将视频保存到收藏夹，以便轻松返回有价值的内容。

#### 验收标准

1. 当用户点击视频上的收藏按钮时，平台应将该视频添加到用户的收藏集合
2. 当用户访问其收藏夹时，平台应按时间倒序显示所有收藏的视频
3. 当用户点击已收藏视频上的收藏按钮时，平台应将其从收藏夹中删除
4. 当视频从平台删除时，平台应将其从所有用户的收藏集合中删除

### 需求 16：内容互动 - 带笔记的标记

**用户故事：** 作为用户，我想标记视频并添加个人笔记，以便捕获关键见解供将来参考。

#### 验收标准

1. 当用户点击标记按钮时，平台应提示用户输入笔记
2. 当用户保存带有笔记的标记时，平台应存储视频引用、笔记文本和时间戳
3. 当用户访问其标记时，平台应显示所有标记的视频及相关笔记
4. 当用户编辑标记笔记时，平台应更新存储的笔记文本
5. 当用户删除标记时，平台应删除标记和相关笔记


### 需求 17：内容互动 - 点赞

**用户故事：** 作为用户，我想点赞视频，以便表示赞赏并帮助提升优质内容。

#### 验收标准

1. 当用户点击点赞按钮时，平台应将视频的点赞数增加一
2. 当用户点击已点赞视频上的点赞按钮时，平台应将点赞数减少一
3. 当用户访问其点赞的视频时，平台应显示用户已点赞的所有视频
4. 当显示视频时，平台应显示总点赞数

### 需求 18：内容互动 - 评论

**用户故事：** 作为用户，我想评论视频并提及同事，以便参与讨论和分享见解。

#### 验收标准

1. 当用户点击评论按钮时，平台应显示评论输入界面
2. 当用户提交评论时，平台应存储评论文本、作者和时间戳
3. 当用户在评论中输入"@"时，平台应显示同事选择器
4. 当用户提及同事时，平台应通知被提及的用户
5. 当用户查看评论时，平台应按时间顺序显示它们及作者信息


### 需求 19：内容分享

**用户故事：** 作为员工用户，我想通过企业微信与同事分享企业内部视频，以便推荐有价值的知识内容和促进企业文化传播。

#### 验收标准

1. 当用户点击分享按钮时，平台应显示包括企业消息传递集成的分享选项
2. 当用户选择企业消息传递时，平台应打开企业消息传递应用，并预填充包含视频链接的消息
3. 当接收者打开共享链接时，平台应直接导航到共享的视频
4. 当视频被分享时，平台应增加视频的分享计数

### 需求 20：离线视频下载

**用户故事：** 作为用户，我想下载视频以供离线观看，以便在没有网络连接的情况下学习。

#### 验收标准

1. 当用户点击下载按钮时，平台应开始将视频下载到本地设备存储
2. 当下载正在进行时，平台应显示下载进度
3. 当用户访问已下载的视频时，平台应显示所有本地缓存的视频
4. 当用户播放已下载的视频时，平台应从本地存储播放而无需网络访问
5. 当设备存储不足时，平台应显示错误消息并阻止下载


### 需求 21：内容举报

**用户故事：** 作为员工用户，我想举报不当内容，以便企业内部平台保持内容质量和企业文化标准。

#### 验收标准

1. 当用户点击举报按钮时，平台应显示预定义的举报原因类别
2. 当用户选择举报原因并提交时，平台应创建包含视频、举报者、原因和时间戳的举报记录
3. 当举报提交时，平台应通知管理员
4. 当用户举报内容时，平台应向用户确认举报提交

### 需求 22：基于专题的内容集合

**用户故事：** 作为用户，我想探索按专题组织的内容，以便全面学习特定主题。

#### 验收标准

1. 当用户访问专题时，平台应显示所有活动的专题集合
2. 当用户选择专题时，平台应显示与该专题相关的所有内容项目
3. 当管理员创建专题时，平台应允许将多个内容项目分配给该专题
4. 当内容添加到专题时，平台应保持管理员定义的专题内容排序


### 需求 23：基于合集的内容集合

**用户故事：** 作为用户，我想访问按合集组织的内容，以便遵循结构化的学习序列。

#### 验收标准

1. 当用户访问合集时，平台应显示所有活动的合集集合
2. 当用户选择合集时，平台应按定义的序列显示内容项目
3. 当用户完成合集中的视频时，平台应自动建议序列中的下一个视频
4. 当管理员创建合集时，平台应强制执行内容项目的顺序排序

### 需求 24：个性化学习计划

**用户故事：** 作为员工用户，我想要根据我的岗位角色定制的学习计划，以便系统地学习企业内部知识和发展相关技能。

#### 验收标准

1. 当用户访问学习计划时，平台应根据用户的角色和技能标签生成计划
2. 当生成学习计划时，平台应包括与用户角色相匹配的相关专题和合集
3. 当用户查看其学习计划时，平台应按优先级顺序显示推荐内容
4. 当用户完成学习计划中的内容时，平台应更新计划进度


### 需求 25：学习提醒

**用户故事：** 作为用户，我想收到关于学习计划的提醒，以便保持一致的学习习惯。

#### 验收标准

1. 当用户启用学习提醒时，平台应根据用户的首选频率安排通知
2. 当提醒时间到达时，平台应向用户的设备发送推送通知
3. 当用户点击提醒通知时，平台应导航到学习计划视图
4. 当用户禁用提醒时，平台应取消所有计划的学习通知

### 需求 26：学习进度跟踪

**用户故事：** 作为用户，我想跟踪我的学习进度，以便监控我的发展并保持动力。

#### 验收标准

1. 当用户访问进度跟踪时，平台应显示包括已观看视频、剩余视频和完成百分比的指标
2. 当用户完成视频时，平台应更新已观看计数并重新计算完成百分比
3. 当用户查看特定专题或合集的进度时，平台应显示该集合的完成状态
4. 当用户完成学习计划中的所有内容时，平台应将计划标记为完成


### 需求 27：用户个人资料管理

**用户故事：** 作为用户，我想查看和管理我的个人资料信息，以便准确地个性化我的内容和推荐。

#### 验收标准

1. 当用户访问其个人资料时，平台应显示用户的头像、姓名、岗位、部门和KOL状态
2. 当用户更新其头像时，平台应验证图片格式和大小，然后更新存储的头像
3. 当用户的角色或部门在企业系统中更改时，平台应同步更新的信息
4. 当用户被指定为KOL时，平台应在其个人资料和内容上显示KOL徽章

### 需求 28：内容发布管理

**用户故事：** 作为员工创作者，我想管理所有已发布和待处理的内容，以便跟踪我在企业内部的创作贡献及其状态。

#### 验收标准

1. 当创作者访问"我的发布"时，平台应显示已发布、草稿、审核中和已驳回内容的选项卡
2. 当创作者选择状态选项卡时，平台应过滤并仅显示具有该状态的内容
3. 当创作者查看已驳回的内容时，平台应显示管理员提供的拒绝原因
4. 当创作者选择草稿或已驳回的项目时，平台应允许编辑和重新提交


### 需求 29：用户活动历史

3. 当用户访问"我的点赞"时，平台应显示所有点赞的视频
4. 当用户访问"我的标记"时，平台应显示所有标记的视频及相关笔记
5. 当用户访问"我的下载"时，平台应显示所有本地缓存的视频

### 需求 30：关注管理

2. 当用户选择关注的创作者时，平台应导航到该创作者的个人资料
3. 当用户从关注列表中取消关注创作者时，平台应从列表中删除该创作者
4. 当用户查看其关注列表时，平台应显示每个创作者的头像、姓名和关注者数量


### 需求 31：创作者个人资料显示

**用户故事：** 作为员工用户，我想查看详细的创作者同事个人资料，以便了解企业内部内容创作者及其专业知识。

#### 验收标准

2. 当用户查看创作者个人资料时，平台应显示该创作者的所有已发布视频
3. 当用户查看创作者个人资料时，平台应显示创作者的关注者数量和关注数量
4. 当创作者具有KOL指定时，平台应在其个人资料上突出显示KOL徽章

### 需求 32：创作者关注和联系

**用户故事：** 作为员工用户，我想关注创作者同事并通过企业微信联系他们，以便了解他们的内容更新并提出问题。

1. 当用户查看创作者个人资料时，平台应显示关注按钮

3. 当用户点击创作者个人资料上的联系按钮时，平台应打开企业消息传递应用并与该创作者进行对话
4. 当用户已经关注创作者时，平台应显示取消关注按钮


### 需求 33：学习分析

2. 当计算观看时间时，平台应汇总所有已完成视频的时长
3. 当用户查看分析时，平台应按内容分类或专题显示细分
4. 当用户保持连续学习天数时，平台应计算并显示学习连续天数

### 需求 34：排行榜和游戏化

**用户故事：** 作为员工用户，我想看到排行榜和成就，以便激励我参与企业内部学习和创作，丰富企业文化氛围。

#### 验收标准

2. 当计算排名时，平台应每天更新排行榜位置
3. 当用户达到里程碑时，平台应授予徽章或成就
4. 当用户查看其个人资料时，平台应显示获得的徽章和成就


### 需求 35：通知偏好设置

**用户故事：** 作为用户，我想控制通知设置，以便接收相关警报而不会被淹没。

#### 验收标准

1. 当用户访问通知设置时，平台应显示学习提醒、审核通知和互动通知的开关
2. 当用户禁用通知类型时，平台应停止发送该类型的通知
3. 当用户启用通知类型时，平台应恢复发送该类型的通知
4. 当启用互动通知且有人点赞、评论或分享用户的内容时，平台应发送通知

### 需求 36：视频质量设置

**用户故事：** 作为用户，我想控制视频质量，以便在观看体验和数据使用之间取得平衡。

#### 验收标准

1. 当用户访问视频质量设置时，平台应显示自动、高清和标清的选项
2. 当用户选择自动质量时，平台应根据网络条件调整视频质量
3. 当用户选择高清时，平台应以最大可用质量流式传输视频
4. 当用户选择标清时，平台应以降低的质量流式传输视频以节省带宽
5. 当播放期间网络条件恶化时，平台应缓冲内容以防止中断


### 需求 37：缓存管理

**用户故事：** 作为用户，我想管理缓存内容，以便在需要时释放设备存储空间。

#### 验收标准

1. 当用户访问缓存管理时，平台应显示缓存内容的总大小
2. 当用户选择清除缓存时，平台应删除所有本地存储的视频文件
3. 当缓存被清除时，平台应将显示的缓存大小更新为零
4. 当用户确认清除缓存时，平台应删除已下载的视频但保留用户数据和偏好设置

### 需求 38：后台内容管理

### 需求 39：后台内容上传

**用户故事：** 作为企业管理员，我想直接通过后台上传内容，以便高效发布企业精选材料和官方内容。

### 需求 41：精选内容推广

**用户故事：** 作为企业管理员，我想在首页推荐高质量内容，以便员工发现有价值的学习材料和企业文化内容。

### 需求 42：内容审核工作流

**用户故事：** 作为企业管理员，我想在发布前审核员工提交的内容，以便只有符合企业标准和文化的内容到达员工用户。

4. 当管理员拒绝内容时，后台系统应要求拒绝原因并将状态更改为"已驳回"
5. 当管理员执行批量审核时，后台系统应使用所选操作处理所有选定的项目

3. 当专家批准内容时，后台系统应更新状态并继续发布
